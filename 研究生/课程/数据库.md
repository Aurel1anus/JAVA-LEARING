微分编程：其核心思想是：**让计算机程序本身变得可微分**，从而我们可以直接对程序进行求导，并利用梯度信息来优化程序的参数。
```
假设我们有一个简单的程序：`y = a * x + b`

- **传统编程**：
    
    - **目标**：给定输入 `x` 和固定的参数 `a`, `b`，计算出输出 `y`。
        
    - **过程**：`x=2, a=3, b=1` → 程序执行 → `y=7`
        
- **微分编程**：
    
    - **目标**：我们有一组观测数据 `(x₁, y₁), (x₂, y₂), ...`，我们希望找到参数 `a` 和 `b`，使得程序 `y = a * x + b` 的输出最符合这些数据（即，解决一个**线性回归**问题）。
        
    - **过程**：
        
        1. 随机初始化参数 `a` 和 `b`。
            
        2. 将数据 `x` 输入程序，得到预测值 `y_pred`。
            
        3. 计算损失函数，例如 `L = (y_true - y_pred)²`。
            
        4. **关键步骤**：使用**自动微分**，计算损失 `L` 关于参数 `a` 和 `b` 的梯度 `(∂L/∂a, ∂L/∂b)`。这告诉我们为了减小误差，`a` 和 `b` 应该朝哪个方向调整。
            
        5. 根据梯度更新参数：`a = a - α * ∂L/∂a`，`b = b - α * ∂L/∂b`（其中 `α` 是学习率）。
            
        6. 重复步骤2-5，直到损失足够小。
```
